# noncanon

> local-first collaborative worldbuilding

canon is what you choose to pull in. everything else is noncanon — until it isn't.

```toml
[dependencies]
noncanon-core = "0.1"
```

## what it does

noncanon is a library for building worldbuilding tools where:

- the world lives on your machine, not a platform
- canon is a local concept — you decide what's true in your copy
- divergence is a feature — parallel canons are fine
- you don't need the whole world, just the parts you're building on

## what it doesn't do

rendering, social features, wikis, conflict resolution UIs. those are separate concerns. noncanon handles the data model and canon semantics.
